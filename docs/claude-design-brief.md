# love-train 小程序 · Claude Design 改造 Brief

## 产品语境

**love-train** 是给男性的情感咨询 AI（童锦程风格诊断 + 行动方案）。微信小程序版本，**体验版定位**：让用户当场把困扰发过来——文字 / 图片（聊天截图 / 对方主页），AI 回一段口语化的诊断 + 下一步怎么做。

调性关键词：**直给、不啰嗦、像兄弟在跟你说话**。所以 UI 也必须跟得上文风——干净、克制、不要花哨。

---

## 当前状态（基线）

**3 个页面**：

### 1. 登录页
- 中心放一个衬线大写 `l`（serif italic 风格预留）
- 主标题 `love-train`
- 一句引导文案：「兄弟，遇到什么情况了，发给我看看。」
- 一个微信一键登录按钮 + 用户协议勾选 + 出品方落款
- 现状：基础可用，配色偏冷（`#f5f5f7` 灰底 + 系统蓝按钮），缺辨识度

### 2. 聊天页（核心）
- 顶部：头像 + 昵称 + 今日配额 `4/10` + ··· 菜单
- 中间：消息列表（用户气泡黑底白字、AI 气泡浅灰底深色字）+ 时间戳
- 上传图片缩略图行（选了图但还没发出时显示）
- 底部输入区：`+`（图片）/ textarea（自适应高度，1–5 行）/ 圆形 `↑` 发送按钮（已用 Claude 橙 `#cc785c`）
- 现状：功能闭环，但配色平、字体单一、AI 气泡和用户气泡区分弱

### 3. 关于页
- 应用 logo + 版本号 + 一句产品描述
- 三张卡片：用户协议 / 隐私协议 / 联系方式
- 一个「清空本地聊天记录」按钮 + 落款
- 现状：纯卡片堆叠，缺层次

## 目标语言：Claude Design（极简版）

参考 claude.ai 桌面端的设计基调，**只取以下骨架**，放弃所有花哨：

### 配色（已定，全局复用）

```
--canvas:    #f7f3ec   底色（暖米白）
--card:      #efe9dd   卡片底
--ink:       #1f1f1c   主文字
--ink-mute:  #5d5b56   次文字
--orange:    #cc785c   Claude 橙（主操作色，仅按钮 / 强调 / AI 标签用）
--burgundy:  #8b3a3a   深酒红（仅「危险 / 警告」色，比如清空记录）
```

### 字体

- **正文**：苹方（PingFang SC / 系统默认）—— 不引外部字体
- **品牌点睛**：仅在登录页 `l` / `love-train` 标题处用 serif italic（`Source Serif`），微信内置不到就用 `Songti SC` 兜底，**不引网络字体文件**

### 间距与圆角

- 圆角统一：卡片 16rpx、按钮 24rpx、头像/小图标 50%
- 行高 1.5
- 段落间留白偏宽（24rpx 起）

### 形态原则

- **横线代替阴影**：用 1rpx 浅边框分割区域，不用 box-shadow
- **大留白胜过装饰**：宁可空也不要塞图标
- **一处橙、一处酒红，其余黑白米**：醒目色不超过 2 处，避免视觉吵闹

## 目标输出（按页拆）

### 登录页

- 整页换成米白底
- `love-train` 用 serif italic 居中，下方一句话用 mute 色
- 微信登录按钮：方角不要圆得像药丸，黑底白字（不是橙色——橙色留给「行动」，登录是入口）
- 协议文案 mute，链接下划线不变色

### 聊天页

- 顶栏白底改米白底，去阴影改下边线
- AI 气泡：米白卡片色 `#efe9dd`，左上角加一个橙色小圆点 + 「诊断中」标签（流式输出时显示，结束后隐藏）
- 用户气泡：黑底白字保留，圆角 16rpx
- 输入区：浅卡片色背景 + 现在已经做好的 textarea + 橙色 `↑`
- 配额数字（4/10）：用 mute 色加细横线 `4 / 10`，不要彩色徽章

### 关于页

- 三张卡片：米白底 + 1rpx 边框，标题前一个橙色小圆点（呼应 AI 气泡）
- 「清空聊天记录」用 ghost 按钮：透明底 + 酒红字 + 酒红边框
- 落款 mute 居中

## 性能 / 工程约束（**必须遵守**）

1. **不引任何外部字体**（不 @import google fonts，不 wx.loadFontFace）
2. **不写任何过渡 / 动画**（`transition`、`animation`、`keyframes` 全禁），只用纯状态色切换
3. **不引图片素材**：所有图标用文字符号或 unicode（`+` `↑` `·` `›`）或 1 个 inline SVG
4. **CSS 总量控制**：每页 wxss 不超过 200 行
5. **不改 wxml 结构**：尽量只动 class 名和样式，避免破坏现有数据绑定
6. **首屏字符数据 ≤ 现状**（不能为了好看新增 view 嵌套）

---

## 验收标准

- 三个页面一眼能看出「是 Claude 那种感觉」——米白 + 橙色点睛 + 大留白
- 滑动 / 点击 / 输入零延迟（不引入任何 transition）
- 包体积增量 < 5KB
- 真机老机型（5 年前的中端机）首屏渲染时间不增加
