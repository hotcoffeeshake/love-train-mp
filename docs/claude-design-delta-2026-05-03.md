## 1. about 页 — 删与加

### 删除
- 整段副标题 `给男性的情感咨询助手。把困扰发过来，给你一段不啰嗦的诊断。` —— **去掉**（违禁词风险）。保留 `love-train` 品牌字 + 版本号即可。
- 当前的「会员状态卡」（成块的 member-card）和「邀请好友卡」（invite-card）—— **整块去掉**，迁移到别处（见 §2、§3）。

### 加 / 改
- 在原有「用户协议 / 隐私政策 / 联系我们」三张卡上方，加**一张同样风格的卡** —— 标题「**开通付费版**」，副标题「每日 30 次 · 免广告」，右侧 `›` 箭头。点这张卡跳到「付费页面」（新页，§3）。
- 这张卡是**唯一**的开通入口在 about 页。视觉上和其他三张卡一致（米白底、左侧橙圆点、右侧 `›`），但**圆点用付费版橙色突出一点**。

### Spec 字段建议
- 卡片标题：`开通付费版`（付费用户改显示「**会员中心**」，下方 meta 改「有效期至 YYYY-MM-DD · 续费 ¥20/月」）
- 卡片左侧圆点：`#e76f51`（主橙）

## 2. chat 页 — 顶端显著位置加邀请

### 加
- 在**聊天列表上方、quota-badge 同一行 / 下一行**（你裁），加一个「**邀请条**」：
  - 形态参考：横向胶囊条，米白底 + 橙色 1px 边 + 8rpx 圆角
  - 文案：「**邀请好友 · 双方各 +5 次免费查询 →**」
  - 右侧 `→` 箭头表示可点
  - 点击行为：唤起小程序原生分享（调 `wx.openShareImage` / 直接拉起转发面板）

### 备注
- 这条不展示用户的邀请码 6 位字符。邀请码是分享卡 path 里偷偷带的，用户不需要看到。
- 已经付费的用户也显示这条（邀请奖励对付费用户也加 bonus，多多益善）。

### 文案优先级
- 紧凑版（一行装得下）：`邀请好友 +5 次免费查询 →`
- 宽松版（两行也行）：`邀请好友 · 双方各 +5 次免费查询 →`

## 3. 付费页面 (新建)

### 触发入口
1. about 页「开通付费版」卡 → 跳过来
2. chat 页发消息时如果返回 429（额度用完）→ 弹 modal `今日 10 次已用完，要不要开通付费版？` → 点「**去开通**」跳过来
3. （未来）右上角「···」展开菜单可加一项

### 页面元素（从上到下）

```

│  [← 返回]                            … ⊙ │
│                                          │

│   │  · 每日 10 次免费查询           │    │
│   │  · 含广告                       │    │
```

### 关键视觉
- 两张对比卡（普通 vs 付费）：高度一致，左卡米白边框，右卡橙色高亮边框 + 顶部小标签「推荐」
- 主 CTA 按钮：用 about 页登录按钮同款黑底白字 / 或橙色填充（你定）
- 付费用户进来：右卡顶部小标签从「推荐」变「当前」，主 CTA 变「**续费**」

### 动作
- 主 CTA 点击 → 调 `api.createOrder({months: 1})`
  - mock 模式：modal「已模拟开通付费（测试模式）」→ 关闭后页面刷新成付费态
  - real 模式：唤起 `wx.requestPayment` → 成功后轮询 `/auth/me` 直到 `is_paid=true`

## 4. chat 页额度用完的引导（新增）

### 触发
chat 页发送消息后，后端返回 429 `QUOTA_EXCEEDED`。

### 当前
输入框变灰 + placeholder 「今日 10 次已用完」。

### 改成
弹 modal：
- 标题：`今日额度用完了`
- 内容：`免费版每日 10 次。开通付费版每天 30 次，没有广告。`
- 主按钮：`去开通` → `wx.navigateTo({ url: '/pages/subscribe/subscribe' })`
- 次按钮：`先不了` → 关闭 modal
